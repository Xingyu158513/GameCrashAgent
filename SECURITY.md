# Security Policy

## Supported versions

Security fixes are applied to the latest released minor version.

## Reporting a vulnerability

Do not open a public issue for a vulnerability, leaked credential, or privacy exposure. After the repository is published, use GitHub's private vulnerability reporting or a private Security Advisory.

Include:

- affected version and Windows version;
- the smallest safe reproduction steps;
- expected and actual behavior;
- whether the issue could modify a system, expose report data, or execute unexpected commands.

Do not include real credentials, raw private logs, or another person's data.

## Security boundary

The project is read-only by design. A proposed change that adds system mutation, automatic upload, remote execution, or automatic remediation is security-sensitive and requires explicit maintainer review before implementation.
